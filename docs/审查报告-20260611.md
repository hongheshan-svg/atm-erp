### 主数据(物料/客户/供应商/仓库/信用)（masterdata）

## 五、审查方法说明

本次审查采用多 agent 并行工作流，全程只读源码不修改业务代码。每个业务模块由一个专项 agent 进行前后端对照审查，逐一追踪前端按钮 → API 调用 → 后端 ViewSet / Serializer → 模型/数据库路径，覆盖业务逻辑（logic）、页面流程（flow）、前后端契约（contract）、增删改查（crud）、UI 体验（ui）五个维度。

对 critical/high 级发现，每条由独立的"怀疑者" agent 进行对抗式核实：critical 发现需两票独立核实，high 发现需一票；核实者尝试沿完整调用链复现以驳倒或确认原始结论，部分发现经核实后被修正严重级（correctedSeverity）。

本次共运行 **147 个 agent**，历时约 **2.8 小时**，消耗约 **763 万 token**。

**注**：e2e-flow 横切单元（端到端主流程链路）因模型 opus-4-8-1m 持续降级超时未能产出结果，本报告不含该单元；但其端到端断链结论与各模块 flow 类发现高度重叠，整体结论不受影响。
