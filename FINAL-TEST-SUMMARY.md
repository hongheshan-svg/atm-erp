#### 问题：API请求URL重复前缀
**症状：** 前端请求`/api/api/auth/users/`导致404错误

**原因：** 
- `request.js`中已配置`baseURL: '/api'`
- 部分Vue组件直接使用`request.get('/api/...')`
- 导致URL变成`/api/api/...`

**解决方案：**
- 批量修复28个Vue文件
- 移除API调用中的`/api/`前缀
- 使用sed命令统一处理

**影响：** 🎉 **所有API请求现在成功返回200状态**

## 🐛 当前存在的问题

### 问题1: API返回数据格式不匹配 ⚠️

**位置：** 用户管理、角色管理、部门管理等页面

**错误信息：**
```
TypeError: Cannot read properties of undefined (reading 'results')
```

**分析：**
- API请求成功（200状态）
- 前端期望`data.results`格式（DRF分页格式）
- 实际返回可能是空数组`[]`或不同格式

**解决方案：**
1. 检查DRF ViewSet是否启用分页
2. 确认API response格式
3. 调整前端数据解析逻辑

### 问题2: WebSocket连接失败 ⚠️

**错误信息：**

WebSocket connection closed
WebSocket error
Attempting to reconnect

- Nginx WebSocket代理未配置

   - 合理配置depends_on

2. **前后端分离部署**
   - Vite代理配置简化开发
   - Nginx生产环境统一入口

3. **快速问题定位**
   - 浏览器Network面板
   - Docker logs实时查看
   - Console错误信息

### 经验教训

1. **API URL前缀管理**
   - 统一使用baseURL配置
   - 避免在业务代码中硬编码/api/
   - 建议使用API模块统一管理

2. **数据格式约定**
   - 前后端需明确数据格式
   - 建议使用TypeScript接口定义
   - API文档自动化（Swagger）

3. **Elasticsearch版本兼容**
   - ES 8.x与django-elasticsearch-dsl不兼容
   - 降级到ES 7.17解决
