# ERP编码系统总览

## 系统架构

ERP系统中有**两套**编码管理系统，分别适用于不同的业务场景：

### 1. 通用编码规则系统（系统管理 > 编码规则）

**适用范围**：大部分业务单据的编号生成

**特点**：
- 灵活配置前缀、日期格式、序列号
- 支持多种重置模式（不重置/每日/每月/每年）
- 统一的管理界面
- 自动序列号累加

**支持的单据类型**：
- ✅ 项目编号
- ✅ 采购合同
- ✅ 销售合同
- ✅ 采购申请
- ✅ 采购订单
- ✅ 销售订单
- ✅ 销售报价
- ✅ 发货单
- ✅ 发票
- ✅ 收货单
- ✅ 库存移动
- ✅ 库存调整

**配置位置**：系统管理 > 编码规则

**使用文档**：`CODING_RULES_GUIDE.md`

---

### 2. 物料编码专用系统（基础数据 > 物料管理）

**为什么单独处理？**

物料编码有非常特殊的业务规则，不适合使用通用编码规则系统：

1. **复杂的条件逻辑**
   - 一级代码决定年份：有图=当前年份，无图=99
   - 二级代码有8种固定分类

2. **特殊的编码结构**
   - 一级代码(1位) + 二级代码(1位) + 年份(2位) + 流水号(6位)
   - 总长度固定为10位

3. **业务语义强**
   - 编码本身包含重要的分类信息
   - 不同类别需要独立的流水号序列

**编码规则**：
```
格式：一级代码(1位) + 二级代码(1位) + 年份(2位) + 流水号(6位)

一级代码：
  1 = 有图
  2 = 无图

二级代码：
  1 = 机加
  2 = 钣金
  3 = 特殊工艺
  4 = 其他
  5 = 机械类
  6 = 电气类
  7 = 耗材辅料
  8 = 办公用品

年份：
  有图 → 当前年份后两位（如25）
  无图 → 固定为99

流水号：
  6位数字，根据前缀自动累加（000001-999999）
```

**使用方式**：
1. 进入：基础数据 > 物料管理
2. 点击"添加物料"
3. 在"物料编码"输入框右侧点击"生成"按钮
4. 选择一级代码和二级代码
5. 点击"生成编码"→"应用"

**使用文档**：`ITEM_CODE_RULE.md`

## 如何选择编码系统？

### 使用通用编码规则系统
✅ 编码格式相对简单
✅ 前缀+日期+序列号的标准结构
✅ 不需要复杂的条件判断
✅ 需要统一管理和配置

### 使用物料编码专用系统
✅ 物料编码（已实现）
✅ 其他有特殊业务逻辑的编码

## 集成指南

### 如何让业务模块使用通用编码规则？

参考项目编号的实现方式：

```python
# 在模型的save方法中
from apps.core.utils import generate_code

class YourModel(BaseModel):
    code = models.CharField(max_length=50, unique=True, blank=True)
    
    def save(self, *args, **kwargs):
        if not self.code:
            # 使用编码规则生成
            self.code = generate_code('DEFAULT_PREFIX', rule_type='YOUR_RULE_TYPE')
        super().save(*args, **kwargs)
```

### 如何创建特殊编码系统？

参考物料编码的实现方式：

1. **创建专用生成器**（如 `item_code_generator.py`）
   - 实现业务逻辑
   - 确保线程安全（使用 `select_for_update()`）
   - 提供API接口

2. **创建前端界面**
   - 可视化参数选择
   - 实时预览
   - 一键应用

3. **编写文档**
   - 规则说明
   - 使用方法
   - 示例展示

## 常见问题

### Q: 为什么物料编码不在编码规则管理中？
A: 物料编码有特殊的业务逻辑（如根据一级代码决定年份），使用专用系统更合适。

### Q: 可以修改物料编码规则吗？
A: 物料编码规则由业务部门定义，已在代码中固化。如需修改，请联系开发团队。

### Q: 其他编码类型如何集成？
A: 大部分编码使用通用编码规则系统。只有特殊业务逻辑的编码才需要专用系统。

### Q: 编码冲突怎么办？
A: 两套系统都使用数据库行锁确保编码唯一性，不会产生冲突。
