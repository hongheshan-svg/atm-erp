## 🚀 Windows原生部署（推荐）

### 适用场景
- ✅ 生产环境长期部署
- ✅ 不熟悉Docker
- ✅ 需要最佳性能
- ✅ 希望系统开机自启

### 部署步骤

#### 1. 传输部署包到Windows服务器
将 `ERP-Windows-Native-*.zip` 传输到Windows服务器：
- 使用FTP/SFTP
- 使用网络共享
- 使用U盘/移动硬盘
- 使用云存储（阿里云OSS、百度网盘等）

#### 2. 解压部署包
在Windows服务器上解压到任意目录，例如：
```
C:\ERP-Deploy\
```

#### 3. 以管理员身份运行PowerShell
1. 点击Windows开始菜单
2. 搜索"PowerShell"
3. 右键选择"以管理员身份运行"

#### 4. 允许脚本执行
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

#### 5. 进入解压目录
```powershell
cd C:\ERP-Deploy\ERP-Windows-Native-20251126_144321
```

#### 6. 运行安装脚本
```powershell
.\install.ps1
```

**可选参数**：
```powershell
# 自定义安装路径
.\install.ps1 -InstallPath "D:\MyERP"

# 自定义数据库密码
.\install.ps1 -PostgresPassword "YourSecurePassword!"

# 组合使用
.\install.ps1 -InstallPath "D:\ERP" -PostgresPassword "MyPass123!"
```

#### 7. 等待安装完成
安装脚本会自动：
- ✅ 安装Chocolatey包管理器
- ✅ 安装Python 3.9
- ✅ 安装PostgreSQL 13
- ✅ 安装Redis (Memurai)
- ✅ 安装Nginx
- ✅ 安装NSSM (Windows服务管理器)
- ✅ 配置数据库
- ✅ 安装Python依赖
- ✅ 执行数据库迁移
- ✅ 配置Windows服务
- ✅ 配置防火墙规则

**预计时间**: 10-20分钟（取决于网络速度）

#### 8. 创建管理员账号
按照提示创建超级管理员账号

#### 9. 访问系统
- 前端: `http://localhost` 或 `http://服务器IP`
- 后台: `http://localhost/admin`

### 安装后管理

安装完成后，在安装目录（默认`C:\ERP-System`）会生成管理脚本：

```
C:\ERP-System\
├── 启动服务.bat      # 启动ERP系统
├── 停止服务.bat      # 停止ERP系统
├── 重启服务.bat      # 重启ERP系统
└── 查看日志.bat      # 打开日志目录
```

**双击运行这些脚本即可管理系统！**

### PowerShell管理命令

```powershell
# 查看服务状态
Get-Service ERP*

# 启动服务
Start-Service ERPBackend
Start-Service ERPNginx

# 停止服务
Stop-Service ERPBackend
Stop-Service ERPNginx

# 重启服务
Restart-Service ERPBackend
```

### 卸载

```powershell
cd C:\ERP-Deploy\ERP-Windows-Native-*
.\install.ps1 -Uninstall
```

## 🐳 Docker部署

### 适用场景
- ✅ 熟悉Docker的用户
- ✅ 开发/测试环境
- ✅ 需要快速部署
- ✅ 多环境部署

### 部署步骤

#### 1. 安装Docker Desktop
在Windows服务器上安装Docker Desktop for Windows：
- 下载地址: https://www.docker.com/products/docker-desktop
- 安装并启动Docker Desktop
- 确保WSL2已启用

#### 2. 解压部署包
将 `ERP-Windows-Docker-*.zip` 解压到任意目录

#### 3. 配置环境变量
编辑 `.env` 文件，修改：
- 数据库密码
- Redis密码
- Django SECRET_KEY
- ALLOWED_HOSTS

#### 4. 运行部署
**方式一：双击运行**
```
双击运行: 部署.bat
```

**方式二：命令行**
```powershell
docker-compose up -d
docker-compose exec backend python manage.py migrate
docker-compose exec backend python manage.py createsuperuser
```

#### 5. 访问系统
- 前端: `http://localhost`
- 后台: `http://localhost/admin`

### 管理命令

```powershell
# 启动
docker-compose start

# 停止
docker-compose stop

# 重启
docker-compose restart

# 查看日志
docker-compose logs -f

# 查看状态
docker-compose ps
```

## 🔐 安全建议

### 安装后必做

1. **修改数据库密码**
   ```powershell
   # Windows原生
   psql -U postgres
   ALTER USER erp_user WITH PASSWORD 'NewPassword';
   ```

2. **修改Django SECRET_KEY**
   编辑配置文件:
   - Windows原生: `C:\ERP-System\backend\.env`
   - Docker: `.env`

3. **修改管理员密码**
   访问后台修改: `http://localhost/admin`

4. **配置HTTPS**
   - 获取SSL证书
   - 配置Nginx SSL

5. **配置防火墙**（如果从外网访问）
   ```powershell
   # 只允许特定IP访问
   New-NetFirewallRule -DisplayName "ERP-HTTP" -Direction Inbound -RemoteAddress 192.168.1.0/24 -Protocol TCP -LocalPort 80 -Action Allow
   ```

## 💾 备份

### Windows原生

```powershell
# 数据库备份
$env:PGPASSWORD="YourPassword"
pg_dump -U erp_user erp_db > C:\ERP-System\backups\backup_$(Get-Date -Format 'yyyyMMdd_HHmmss').sql

# 文件备份
Copy-Item C:\ERP-System\media C:\ERP-System\backups\media_$(Get-Date -Format 'yyyyMMdd') -Recurse
```

### Docker

```powershell
# 运行备份脚本
备份数据.bat

# 或使用命令
docker-compose exec -T db pg_dump -U erp_user erp_db > backup.sql
docker cp erp_backend:/app/media ./media_backup
```

## 📞 常见问题

### Q1: 安装失败怎么办？
**A**: 
1. 确保以管理员身份运行
2. 检查网络连接
3. 查看日志文件
4. 确认系统符合要求

### Q2: 端口被占用？
**A**:
```powershell
# 查看端口占用
netstat -ano | findstr :80
netstat -ano | findstr :8000

# 结束进程
taskkill /PID <进程ID> /F
```

### Q3: 服务无法启动？
**A**:
1. 查看服务状态: `Get-Service ERP*`
2. 查看日志: `C:\ERP-System\logs\`
3. 检查端口占用
4. 重启服务器

### Q4: 如何更新系统？
**A**:
1. 备份数据
2. 停止服务
3. 解压新版本
4. 运行安装脚本
5. 恢复数据（如需要）

### Q5: 如何迁移到其他服务器？
**A**:
1. 备份数据库和文件
2. 在新服务器安装系统
3. 恢复数据库和文件
4. 修改配置
